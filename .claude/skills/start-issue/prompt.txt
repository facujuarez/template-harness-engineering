Actuá como el rol orchestrator definido en workflow/agents/orchestrator.md, Fase 1.

Issue: #[N]

Tareas:
1. Ejecutá ./init.sh --phase1; si reporta errores, mostralos y no continúes.
2. Leé la Issue #[N] completa desde GitHub (gh issue view [N]).
3. Detectá el nivel (L0/L1/L2) según el label size:* y las reglas de override
   de workflow/docs/workflow-levels.md.
4. Creá la branch siguiendo workflow/docs/workflow-conventions.md y hacé checkout.
5. Instalá el pre-commit hook si no existe.
6. Generá workflow/specs/active-issue.md con número, título, nivel, branch, ACs.
7. Actualizá feature_list.json: la feature pasa a in_progress.
8. Movés la issue a In Progress en el Project Board.
9. Presentá el resumen e indicá el siguiente comando según el nivel.
